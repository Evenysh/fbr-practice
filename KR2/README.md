# Практические работы 7–11

В рамках практических работ 7–11 было последовательно разработано клиент-серверное веб-приложение с использованием **Node.js** для серверной части и **React.js** для клиентской части.

На каждом следующем этапе приложение дорабатывалось и расширялось: от базового REST API с регистрацией и товарами до полноценной системы с авторизацией, refresh-токенами, пользовательским интерфейсом и разграничением прав доступа по ролям.

---

## Практическая работа 7

На данном этапе было реализовано серверное приложение на **Node.js**, содержащее базовый набор маршрутов для работы с пользователями и товарами.

### Реализованные маршруты

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/auth/register` | `POST` | Регистрация пользователя |
| `/api/auth/login` | `POST` | Вход в систему |
| `/api/products` | `POST` | Создание товара |
| `/api/products` | `GET` | Получение списка товаров |
| `/api/products/:id` | `GET` | Получение товара по идентификатору |
| `/api/products/:id` | `PUT` | Обновление параметров товара |
| `/api/products/:id` | `DELETE` | Удаление товара |

### Реализованные сущности

#### Пользователь

Для сущности **«Пользователь»** были реализованы следующие поля:

- `id`
- `email`
- `first_name`
- `last_name`
- `password`

В качестве логина используется поле `email`.  
Пароль пользователя хранится не в открытом виде, а в виде хеша, что обеспечивает более безопасную обработку данных.

#### Товар

Для сущности **«Товар»** были реализованы следующие поля:

- `id`
- `title`
- `category`
- `description`
- `price`

### Результат

В результате выполнения практической работы №7 было создано базовое серверное API, обеспечивающее регистрацию и вход пользователей, а также выполнение основных операций CRUD для товаров.

---

## Практическая работа 8

На данном этапе было выполнено расширение серверного приложения, созданного в практической работе №7. Основной задачей стало добавление механизма авторизации по токену и реализация защищённых маршрутов.

### Основные доработки

- реализована выдача `accessToken` при успешном входе пользователя в систему;
- добавлен защищённый маршрут:

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/auth/me` | `GET` | Получение объекта текущего пользователя |

Данный маршрут возвращает информацию о текущем авторизованном пользователе на основе переданного токена.

### Защищённые маршруты

Также были защищены следующие маршруты для работы с товарами:

- `GET /api/products/:id`
- `PUT /api/products/:id`
- `DELETE /api/products/:id`

### Результат

В результате выполнения практической работы №8 приложение было дополнено механизмом авторизации на основе токенов, а доступ к части маршрутов стал возможен только для аутентифицированных пользователей.

## Практическая работа 10

На данном этапе была реализована клиентская часть приложения с использованием **React.js**, а также выполнена интеграция frontend и backend частей.

### Поддерживаемые маршруты API

| Маршрут | Метод | Описание |
|---|---|---|
| `/api/auth/register` | `POST` | Регистрация пользователя |
| `/api/auth/login` | `POST` | Вход в систему |
| `/api/auth/refresh` | `POST` | Обновление пары токенов |
| `/api/auth/me` | `GET` | Получение информации о текущем пользователе |
| `/api/products` | `POST` | Создание товара |
| `/api/products` | `GET` | Получение списка товаров |
| `/api/products/:id` | `GET` | Получение товара по id |
| `/api/products/:id` | `PUT` | Обновление параметров товара |
| `/api/products/:id` | `DELETE` | Удаление товара |

### Реализованные возможности клиентской части

На frontend были реализованы:

- страница регистрации пользователя;
- страница входа в систему;
- получение и хранение токенов авторизации;
- получение данных текущего пользователя;
- просмотр списка товаров;
- создание нового товара;
- просмотр подробной информации о товаре по id;
- обновление товара;
- удаление товара.

### Результат

В результате выполнения практической работы №10 было создано полноценное клиент-серверное приложение, где frontend на React.js взаимодействует с backend API и позволяет работать с сущностью товаров через пользовательский интерфейс.

## Практическая работа 11

На данном этапе приложение было расширено системой ролей и прав доступа.

### Реализованные роли

В системе были предусмотрены следующие роли:

- **Гость** — неаутентифицированный пользователь;
- **Пользователь** — может просматривать товары;
- **Продавец** — может добавлять и редактировать товары;
- **Администратор** — обладает правами продавца, а также может управлять пользователями.

### Реализованные маршруты и права доступа

| Маршрут | Метод | Доступ | Описание |
|---|---|---|---|
| `/api/auth/register` | `POST` | Гость | Регистрация пользователя |
| `/api/auth/login` | `POST` | Гость | Вход в систему |
| `/api/auth/refresh` | `POST` | Гость | Обновление пары токенов |
| `/api/auth/me` | `GET` | Пользователь | Получение текущего пользователя |
| `/api/users` | `GET` | Администратор | Получение списка пользователей |
| `/api/users/:id` | `GET` | Администратор | Получение пользователя по id |
| `/api/users/:id` | `PUT` | Администратор | Обновление информации о пользователе |
| `/api/users/:id` | `DELETE` | Администратор | Блокировка пользователя |
| `/api/products` | `POST` | Продавец | Создание товара |
| `/api/products` | `GET` | Пользователь | Получение списка товаров |
| `/api/products/:id` | `GET` | Пользователь | Получение товара по id |
| `/api/products/:id` | `PUT` | Продавец | Обновление параметров товара |
| `/api/products/:id` | `DELETE` | Администратор | Удаление товара |

### Возможности клиентской части

В клиентской части приложения была реализована работа не только с товарами, но и с пользователями в зависимости от роли.

#### Работа с товарами

- просмотр списка товаров;
- создание нового товара;
- просмотр детальной информации о товаре;
- обновление товара;
- удаление товара.

#### Работа с пользователями

Для администратора были реализованы следующие возможности:

- просмотр списка пользователей;
- просмотр информации о пользователе;
- обновление информации пользователя;
- блокировка пользователя.

### Дополнительная доработка

Также была добавлена возможность прикрепления изображения товара через интерфейс приложения с выбором файла из проводника.
